# Code of Conduct

## Our Commitment

We want Pepenium to be a welcoming, respectful and productive project for everyone involved.

Contributors, maintainers and community members are expected to interact in ways that support collaboration, curiosity and constructive feedback.

## Expected Behavior

Examples of behavior that helps keep the project healthy:

- Be respectful and professional
- Assume good intent while still giving honest feedback
- Focus on ideas, code and outcomes instead of personal attacks
- Be open to learning, correction and different levels of experience
- Help keep discussions actionable and relevant to the project

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, intimidation or discrimination
- Insults, personal attacks or hostile language
- Deliberate disruption of discussions, reviews or collaboration
- Publishing private information without permission
- Any conduct that would make participation unsafe or unreasonably difficult

## Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of Conduct.

They may take any action they consider appropriate in response to unacceptable behavior, including warning, moderating, rejecting contributions or removing participation privileges.

## Reporting

If you experience or witness behavior that violates this Code of Conduct, please contact the maintainers privately instead of escalating through a public thread when possible.

Please include:

- What happened
- Where it happened
- Who was involved
- Any relevant links, screenshots or context

All reports will be reviewed in good faith and handled as respectfully and confidentially as practical.

## Project Standard

This Code of Conduct applies across:

- GitHub issues
- Pull requests
- Code review discussions
- Repository-linked project communication
